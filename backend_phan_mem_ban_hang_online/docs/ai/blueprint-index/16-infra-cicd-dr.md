# Blueprint §16 — Infrastructure, environments, CI/CD and disaster recovery

**Source:** `backend_doc/01_BACKEND_ENTERPRISE_IMPLEMENTATION_BLUEPRINT_v2.0.md` §16.1–16.11

- Five environments — local, test/CI, staging, optional preprod, production — with fully separated credentials/network/data; raw production PII is never copied to local/staging (§16.1).
- Production topology baseline: WAF/CDN/load balancer, ≥2 API instances across failure zones, worker autoscaled by queue depth, scheduler with leader election, managed PostgreSQL HA+PITR, managed Redis HA, AI service private with no public ingress, egress allowlist for provider/model endpoints (§16.2).
- Config is typed and validated at startup; missing/invalid critical config fails fast; a dynamic feature flag can never substitute for an immutable security invariant (§16.3).
- CI pipeline (§16.4): PR stage runs lint/type-check/unit/ephemeral-PG-Redis/migrations-from-zero/integration/OpenAPI-AsyncAPI-lint-and-breaking-change/generated-client-diff/tenant-isolation/SAST/secret-scan/SCA; main branch builds signed immutable OCI images with SBOM/provenance and deploys staging by digest, running migration-compatibility/smoke/contract/E2E/DAST/AI-eval-subset; production promotion is backup-verified, canary 5%→25%→100% based on SLO.
- DB migration policy is expand/contract over three releases: N adds nullable/new table/index concurrently with dual read/write and throttled idempotent backfill while old+new code stay compatible; N+1 switches reads and validates; N+2 enforces constraints and drops old columns only after the rollback window. No destructive migration ships before compatible code is rolled out; large indexes use `CREATE INDEX CONCURRENTLY` (§16.5).
- Deployment: stateless API with graceful drain, workers finish/return their lease before terminating, SSE sends a reconnect hint, readiness flips false before termination, containers run non-root on a pinned digest (§16.6).
- Feature flag types: release, experiment, ops kill switch, tenant entitlement, permission/security policy (never a generic client-controlled flag). Every flag needs an owner, default, environment scope, and cleanup ticket; the server evaluates security/entitlement flags, frontend flags are UI convenience only (§16.7).
- Backup/PITR: automated backup + continuous WAL, RPO ≤5 minutes, 35-day snapshot retention baseline; Redis/queue is not the source of truth (outbox/DB allow replay) (§16.8). A backup that hasn't been restored is not considered usable — at least quarterly and before commercial launch, run a restore drill into an isolated environment and measure actual RPO/RTO (§16.9).
- Disaster recovery is v1 warm/cold standby, not active-active; Backend AI Agent does technical triage, but the Human Owner is the decision authority for any recovery-point choice that accepts data loss (§16.10).
- External dependency resilience: mandatory timeouts (no infinite wait), circuit breaker + bulkhead per provider, retry only idempotent/safe operations, contract adapter test fixtures for webhook/API version drift, quota/rate-limit alerting before exhaustion (§16.11).
