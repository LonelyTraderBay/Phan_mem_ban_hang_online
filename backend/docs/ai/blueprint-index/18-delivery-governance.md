# Blueprint §18 — Delivery governance: Definition of Ready, Done, contract freeze

**Source:** `backend_doc/01_BACKEND_ENTERPRISE_IMPLEMENTATION_BLUEPRINT_v2.0.md` §18.1–18.4

- Definition of Ready (§18.1): actor/use case and business outcome, in/out of scope, business invariants and state transitions, permission/tenant/data classification, OpenAPI/AsyncAPI schema or an explicit "no contract change" confirmation, DB impact (migration/backfill/index), transaction boundary/lock/idempotency, audit/event/telemetry requirements, error codes, acceptance tests including negative/concurrency/security, dependencies ready or mocked, feature flag/rollout/rollback plan, and no open P0 business or security question. A ticket missing a critical item goes `needs-refinement` — the Backend AI Agent must never guess it.
- Definition of Done (§18.2): module boundary/strict typing respected; migration is immutable with correct index/constraint/RLS and passes fresh+upgrade tests; OpenAPI/AsyncAPI updated and lint/breaking-change clean; validation/auth/permission/tenant isolation/field masking in place; idempotency/transaction/audit/outbox present for critical commands; matching unit/integration/contract/concurrency/security tests; logs/traces/metrics plus dashboard/alert if critical; error catalog/examples/mock/generated client updated; feature flag and rollout/rollback note written; CI green with no unresolved Critical/High finding; staging smoke/acceptance passed.
- Contract freeze (§18.3): Backend AI Agent and Frontend AI Agent run in separate sessions/repos with no live chat — all coordination goes through `docs/collaboration/contract-gap-board.md` (operation/schema/enum/error/permission/event entries move `proposed → acked → frozen`) and `docs/collaboration/SIGNOFF_TRACKER.md` for anything with no existing rule or that touches an invariant/security/money. A disagreement is resolved either by applying an existing blueprint/ADR rule directly, or by escalating to the decision queue for the Human Owner — there is no "two leads discuss and agree" step. Contract source merges before or with the implementation skeleton; breaking changes mid-sprint require a board entry with impact/migration and both agents' ack.
- Ownership/RACI (§18.4): Human Owner is Accountable for business-rule correctness, risk acceptance, autopilot/high-risk AI activation, and the irreversible production go/no-go; Backend AI Agent is Accountable for routine architecture/ADR decisions, OpenAPI/AsyncAPI authorship, ERD/migrations, and technical self-verification via the `invariant-reviewer`/`module-test-writer` subagents.
