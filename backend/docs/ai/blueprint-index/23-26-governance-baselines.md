# Blueprint §23–26 — Role matrix, retention, feature flags, readiness gate baselines

**Source:** `backend_doc/01_BACKEND_ENTERPRISE_IMPLEMENTATION_BLUEPRINT_v2.0.md` §23 Default role matrix, §24 Retention baseline, §25 Feature flag catalog baseline, §26 Production readiness gate

- §23 default capability × role matrix (Owner/Admin/Manager/Sales/Warehouse/Analyst/Support-platform) is a seed template — a tenant custom role must never be granted a platform-only permission; cost/profit read and AI configure/activate have no default grant for most roles; the full seed lives in `backend_doc/matrices/permission_matrix.csv`.
- §24 retention baseline by data class: access/application logs 30 days; security/audit logs 365 days; raw webhook payload 30 days (keep hash/metadata longer); normalized messages baseline 24 months; idempotency records 24h–7 days (provider/payment keys longer); outbox published rows 7–30 days; AI raw sensitive trace off by default; export files auto-delete ≤24h; backup/PITR 35-day baseline. Actual values must still satisfy legal/contract/data-region requirements; retention jobs must be observable/idempotent with a legal-hold override.
- §25 baseline feature flag keys: `ai.enabled`, `ai.mode.copilot`/`ai.mode.semi_auto`/`ai.mode.autopilot`, `ai.tool.order_draft.enabled`, `ai.tool.inventory_reserve.enabled`, `ai.provider.<provider>.enabled`, `channel.facebook.enabled`, `conversation.sse.enabled`, `order.confirm.v1`, `payment.manual.enabled`, `returns.enabled`, `analytics.dashboard.v1`, `billing.enforcement.hard`, `support.break_glass.enabled`. Every flag needs an owner, safe default, scope (global/tenant/user), expiry/cleanup ticket, and audit on ops changes.
- §26 production readiness gate, six checklists to self-certify before commercial launch: Architecture/Data (§26.1 — ADR/ERD/RLS/state machines complete, no unresolved dual-write), API/Event (§26.2 — contract lint/compat, outbox/inbox/DLQ/replay, SSE reconnect verified), Security/Privacy (§26.3 — ASVS evidence, tenant isolation suite, pentest Critical/High resolved or risk-accepted), AI (§26.4 — AI-R001…R010 enforced, P0 eval = 0 critical violations, kill switch operational), Reliability/Operations (§26.5 — load/soak headroom, restore/rollback drills, zero unexplained reconciliation mismatch), Release/Supply-chain (§26.6 — signed images/SBOM, rehearsed migration, Human Owner go/no-go recorded). This is the final self-certification checklist feeding the §19.14 P12 go/no-go, not a per-ticket gate.
